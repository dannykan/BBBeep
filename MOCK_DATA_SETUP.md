# Mock 數據設置指南

## 🎯 功能說明

已為車牌 **BBP-2999** 創建了完整的 mock 數據，包括：
- ✅ 收件箱各種狀態的消息（已讀、未讀、不同類型）
- ✅ 發送記錄（用於發送記錄頁面）

## 📋 運行 Seed 腳本

### 步驟 1：運行 Seed

```bash
cd backend
npm run prisma:seed
```

這會創建：
- BBP-2999 用戶（如果不存在）
- 3 個發送者用戶（用於發送消息給 BBP-2999）
- 7 條收件箱消息（3 條未讀，4 條已讀）
- 2 個接收者用戶（用於 BBP-2999 發送的消息）
- 4 條發送記錄

### 步驟 2：登入 BBP-2999 帳號

1. 使用手機號碼：`0912345678`
2. 查看後端終端獲取驗證碼
3. 完成登入

### 步驟 3：查看數據

**收件箱** (`/inbox`)：
- 3 條未讀消息
- 4 條已讀消息
- 包含不同類型：車況提醒、行車安全提醒、讚美感謝

**發送記錄** (`/sent`)：
- 4 條發送記錄
- 顯示發送給誰、是否已讀

## 🔄 重新生成 Mock 數據

如果需要重新生成：

```bash
cd backend
npm run prisma:seed
```

**注意**：這會刪除現有的消息並重新創建，但不會刪除用戶。

## 🧪 測試場景

### 場景 1：查看收件箱
1. 登入 BBP-2999 帳號
2. 點擊「收件匣」
3. 應該看到 7 條消息，其中 3 條未讀

### 場景 2：查看發送記錄
1. 登入 BBP-2999 帳號
2. 在首頁點擊「發送記錄」
3. 應該看到 4 條發送記錄

### 場景 3：查看消息詳情
1. 在收件箱或發送記錄中點擊任意消息
2. 查看完整內容和發送者/接收者信息

## 📝 注意事項

1. **用戶創建**：如果 BBP-2999 用戶已存在，seed 腳本會使用現有用戶
2. **數據清理**：每次運行 seed 會刪除現有消息並重新創建
3. **點數**：BBP-2999 用戶初始點數為 20 點
